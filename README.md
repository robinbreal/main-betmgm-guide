# main-betmgm-guide

本仓库用于归档和发布多个独立的 HTML 页面。页面内容涵盖通用信息、操作指引、参考说明等，不针对任何特定域名或具体网站。仓库仅作为静态页面存放与展示使用。

## 项目简介

这是一个以 HTML 页面归档为主要目的的仓库。每个页面均为独立的静态文件，可直接在浏览器中打开查看。仓库本身不包含后端逻辑、数据库或动态交互功能。

## 目录说明

```
├── README.md          # 本文件
├── index.html         # 仓库首页 / 页面索引（可选）
├── pages/             # 存放所有 HTML 页面
│   ├── page-1.html
│   ├── page-2.html
│   └── ...
└── assets/            # 公共资源（CSS、JS、图片等）
    └── ...
```

- **pages/**：所有归档的 HTML 页面均放置于此目录下，文件名应尽量简短、有意义。
- **assets/**：存放页面可能依赖的样式表、脚本或图片等资源文件，便于统一管理与引用。
- **index.html**：可选，可作为仓库的入口页面，提供页面列表或导航链接。

## 页面归档说明

- 每个页面均为完整的、自包含的 HTML 文件（或引用 `assets/` 中的资源）。
- 页面内容应保持通用、中立，避免包含具体域名、网址、营销承诺或诱导点击内容。
- 页面之间无依赖关系，可独立访问。
- 建议页面标题、描述清晰，便于使用者快速了解页面用途。

## 维护说明

- 如需新增页面，请将 HTML 文件放入 `pages/` 目录，并更新 `index.html`（如有）中的导航链接。
- 修改已有页面时，请保持文件命名与内部引用的一致性。
- 公共资源（CSS/JS/图片）请统一存放在 `assets/` 目录下，避免散落在其他位置。
- 删除页面时，请同步清理相关引用。

## 使用方式

1. 克隆本仓库到本地：
   ```bash
   git clone https://github.com/your-username/main-betmgm-guide.git
   ```
2. 直接在浏览器中打开 `pages/` 下的任意 HTML 文件，或通过 `index.html` 浏览页面索引。
3. 也可通过 GitHub Pages 或其他静态托管服务部署后在线访问。
